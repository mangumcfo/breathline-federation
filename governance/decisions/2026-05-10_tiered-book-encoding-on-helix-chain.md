## Problem

Every book's protocol content (frameworks, vocabulary, agent-callable
APIs) should be public, immutably anchored, and queryable by agents
for education. KDP / Audible monetization on the prose continues.
Encryption alone in a public repo doesn't gate prose — whoever can
decrypt is the consumer. Need capability separation, not obfuscation.

---

## Decision

Encode each book as a four-tier artifact on the Helix chain. Protocol
layer encoded as B35 DNA; prose layer encrypted with anchored
commitments; key release gated by SIX-attested receipts.

| Tier | Contents | Form | Who reads |
|---|---|---|---|
| **L0 — Manifest** | book_id, ISBN, title, framework name, author seal, chapter count, root sha256 of full text, Merkle root of L3 chunks, license | Plaintext on chain | Anyone |
| **L1 — Protocol** | Framework spec, vocabulary, citation index, chapter abstracts (~150w), agent-callable API surface | B35-encoded DNA strings (STIP opcodes); each teaching a tx with `tx_type=protocol, dna_string, dna_hash, parent_tx, chain_hash, signature` | Anyone via Unfold API |
| **L2 — Teaching** | Worked examples, prompt library, sanitized case-study summaries | Generated by vertical renderer (`general` for now) from L1 DNA on demand — not stored verbatim | Anyone via renderer |
| **L3 — Prose** | Full chapter text, chunked | Each chunk AEAD-encrypted (e.g., ChaCha20-Poly1305) with per-book key; ciphertext + per-chunk sha256 + Merkle leaf anchored on chain | Holders of L4 key |
| **L4 — Key** | Per-book symmetric key | Escrowed offline by KM-1176; released by SIX-attested receipt (KDP webhook → mint receipt → unlock), ACX attestation, ARC breath-grant, or operator hand-out | Verified buyers / KM-1176 |

## B47 flow applied to a book

```
Chapter authored (= protocol decision in book form)
   → B35 Encoder: teaching → DNA strings (L1)
   → Vertical renderer: DNA → JSON entities → prose draft (feeds L2 + L3 input)
   → Prose chunked + AEAD-encrypted + Merkle-rooted (L3)
   → L0 manifest assembled (root sha256, Merkle root, license)
   → XRPL / SIX anchor (root + Schnorr sig per attestation.schema.json)
   → Cylinder integration (seal.sh appends; chain_sentinel verifies on next boot)
   → Helix Forge later draws on the thread for cross-book synthesis (e.g., capstone.yaml)
   → Renderer reflects with warmth / cadence on retrieval
```

Honors at every step:

- **K1 Human Primacy** — KM-1176 breath at encode + at key escrow
- **K2 Default-Deny** — no L3 read without attested key release
- **K3 Audit-Immutable** — every step sealed via `seal.sh`; chain replay genesis → tip
- **K4 Constitutional-Validated Extension** — new vertical renderers added under `spec_amendment_review`
- **Stillpoint governor** — gates Helix Forge synthesis runs (cadence TBD per redline)

## What an agent sees in practice

*"What does the FORECAST loop teach?"*

→ Fetch L0 + L1 by `book_id`. Call Unfold API:
`VALIDATE → DECODE → ROUTE → GENERATE → HASH → COMPARE → RETURN`.
Receive framework structure + chapter abstracts as JSON entities.
Every claim hash-linkable to a chain position for citation. L3
ciphertext stays opaque.

*"Give me chapter 4 of Family Education Sovereignty."*

→ Fetch L3 ciphertext (free). Request L4 key. Default-deny: no
SIX-attested receipt → refused. Renderer returns purchase pointer
(KDP / Audible).

## Why Education (Series 4) is the right first instance

- `specs/education/` already exists in plaintext — natural L1
  substrate, no rewriting history
- No KDP manuscript exists yet → encode at authoring time, not
  retrofit
- Education sub-track lives cleanly under the `general` renderer
  initially; an `education` renderer can spawn under K4 amendment
  when Series 4 grows
- Anchor book *Family Education Sovereignty* becomes the first book
  published in this scheme. Series 1 (KDP-live) migrates later if at
  all.

## Open questions for KM-1176 redline (deeper primitives)

1. **Granularity** — one DNA string per chapter, or per teaching unit
   (FORECAST step, gate, action class)?
2. **Vertical renderer for Education** — extend `general`, or spawn
   `education` now?
3. **Stillpoint governor cadence** — per-chapter, per-book,
   per-series synthesis runs?
4. **Key mechanics** — single per-book key vs. per-chapter rotation;
   XRPL escrow vs. operator-only; key half-life
5. **Unfold API home** — `breathline-federation/platform/` plane, or
   a new `helix/` plane (keeps the encoder elite and uncapturable per
   the directive)
6. **B35 author surface** — is encoding only ever invoked by KM-1176,
   or can a vertical renderer round-trip its own DNA back through
   B35 under stillpoint review?

## Non-goals

- Does NOT democratize the encoder. B35 stays single-anchor
  (KM-1176).
- Does NOT propose retroactively encrypting Series 1 (KDP-live).
  Scheme starts at Series 4 forward.
- Does NOT replace the receipt-minter. SIX continues as the public
  verification surface.
